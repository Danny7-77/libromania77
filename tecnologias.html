Documentación de Arquitectura Tecnológica

Informe técnico de los componentes del ecosistema de software implementados en el desarrollo de la plataforma Libromanía.

📁 Componente 1 — Base de Datos

MySQL

Descripción y Justificación: Es el sistema de gestión de bases de datos relacional (RDBMS) utilizado para centralizar la persistencia de la plataforma. Se justificó su elección debido a su robustez, velocidad de procesamiento en consultas indexadas y su perfecta compatibilidad con arquitecturas comerciales, lo que permite almacenar de forma íntegra el catálogo de útiles, usuarios y las bitácoras transaccionales de ventas.

MySQL Workbench

Herramienta para el Diagrama E-R: Entorno de diseño visual empleado para la construcción, normalización y modelado del diagrama Entidad-Relación del sistema. Facilitó la ingeniería en reversa, la definición visual de llaves primarias y foráneas, y la estructuración de las restricciones de integridad referencial antes de la creación física de las tablas.

PSeInt

Fundamentos Algorítmicos: Herramienta de soporte utilizada en la etapa inicial para el diseño lógico de las estructuras de control. A través del pseudocódigo y diagramas de flujo, se modelaron los algoritmos esenciales que posteriormente gobiernan las consultas de base de datos, tales como el descuento matemático del stock de almacén y las condiciones lógicas de facturación.

🧠 Reflexión del Aprendizaje (Base de Datos)

El desarrollo del componente de base de datos me permitió comprender la importancia vital de la normalización de tablas para eliminar la duplicidad de información y evitar anomalías en el sistema. Aprendí a traducir requerimientos lógicos de negocio en relaciones físicas y restricciones funcionales reales mediante el modelado del diagrama Entidad-Relación. Asimismo, asimilé cómo la correcta indexación y estructuración de llaves primarias e intencionalidad de datos impacta directamente en el rendimiento de la aplicación al procesar múltiples consultas en tiempo real.

🌐 Componente 2 — Página Web

HTML5

Descripción Técnica: Lenguaje de marcado de hipertexto implementado como el estándar estructural para maquetar la plataforma. Su función principal consiste en organizar de forma jerárquica y semántica cada sección de la aplicación web, proveyendo los contenedores nativos, elementos de formulario para los clientes, tablas estructuradas de reportería analítica y los nodos esenciales del árbol DOM.

CSS3 (Estilos Propios)

Justificación de Elección: Se optó por el desarrollo de CSS propio y nativo en lugar de un framework como Bootstrap. Esta elección se justifica bajo el criterio de diseño de sistemas de optimizar el rendimiento al eliminar la carga de código externo innecesario, permitiendo un control milimétrico sobre el diseño responsivo mediante Flexbox y garantizando la personalización exacta de los efectos y acabados tridimensionales de las tarjetas.

JavaScript Nativo

Interactividad del Sistema: Lenguaje de programación del lado del cliente utilizado para dotar de dinamismo interactivo a la interfaz. Administra los arreglos de datos del carrito de compras, realiza los cálculos en tiempo real en la vista de usuario y controla la manipulación lógica de clases en el DOM para abrir y cerrar el visor de imágenes expandidas en el módulo de reportes.

Visual Studio Code

Editor de Texto / IDE Utilizado: Entorno de desarrollo seleccionado para centralizar la codificación del software. Su elección se fundamenta en su arquitectura ligera, su potente sistema de depuración integrado y la disponibilidad de extensiones clave que aceleran la productividad en el desarrollo, facilitando la auditoría de sintaxis y la integración limpia entre los archivos del proyecto.

🧠 Reflexión del Aprendizaje (Página Web)

El diseño y codificación del componente web me enseñó a valorar la separación estricta de responsabilidades en el desarrollo de software, comprendiendo que el código espagueti degrada drásticamente la mantenibilidad de una aplicación. Al estructurar CSS propio, asimilé el funcionamiento interno del flujo de renderizado en el navegador y cómo diseñar de forma responsiva para múltiples dispositivos de pantalla. Además, la integración de JavaScript me permitió dominar la manipulación dinámica del árbol DOM y la captura de eventos, transformando una interfaz estática en un sistema transaccional interactivo.
